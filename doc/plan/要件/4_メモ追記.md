# 📄 議事録システム開発仕様書

## 【File 4】人間のリアルタイムメモ追記機能

> 📚 [目次](0_index.md)　｜　[← File 3 自動話者分離](3_話者分離.md)　｜　次 → [File 5 議事録作成](5_議事録作成.md)

本ドキュメントは、会議中にユーザー（人間）がリアルタイムに補足情報やメモをタイムラインへ割り込み追記し、それをAIの文字起こしデータと時間軸で同期・統合するための要件定義・仕様書である。

---

## 1. 機能の目的と全体像

音声認識（Whisper）やローカルLLM（Qwen）の文字起こし整形がどれだけ高速でも、マイクの距離や声量、あるいは「ホワイトボードに書かれた文字」などはローカルAIではキャプチャできない。
本機能は、「人間がその場でチャット感覚で割り込みメモを入れる」ことで、タイムライン上にAIと人間のハイブリッドな文脈を作り出し、会議終了後の最終要約（File 5）の精度を最大化することを目的とする。

```text
[ タイムラインの流れ ]
10:05 [鈴木さん] : 「来期の予算は一応、基本ラインとして100万円で…」  <-- AI生成
10:05 [📝人間メモ] : ★ここで社長から「最大150万まで決済を出す」と口頭指示あり <-- 人間追記
10:06 [佐藤さん] : 「了解しました。では150万を上限にプロトタイプを…」  <-- AI生成

```

## 2. 各コンポーネントの仕様・要件

### 2.1 フロントエンドUI/UX 要件（Quasar）

#### ① メモ入力フォーム

* リアルタイムタイムライン画面の最下部、またはサイドパネルに、常設のメモ入力フォーム（`q-input`）を設置する。
* ユーザーが文字を入力し、`Enter`キーを押すか、送信ボタン（`q-btn` / 紙飛行機アイコン）をクリックすることでタイムラインに即座に挿入される。

#### ② タイムラインへのレンダリング（`q-chat-message`）

* AIによる文字起こしブロックと、人間の手動メモを視覚的に明確に区別してレンダリングすること。
    * **AI文字起こし:** 左側配置、通常の背景色（例：`bg-grey-2`）。
    * **人間メモ:** 右側配置、または独自のカラー（例：`bg-amber-1`）、かつ発言者名部分に「📝人間メモ」や「手動補足」といったラベルとアイコンを表示する。



### 2.2 データ構造とタイムスタンプ同期ロジック

人間がメモを送信した瞬間、**フロントエンド（またはバックエンド）でその瞬間のUnixタイムスタンプをミリ秒単位で取得**し、AIの文字起こしデータ（セグメント）と同じタイムライン配列の中にソート（挿入）する。

#### 【タイムライン要素の統合データ構造案（TypeScript）】

```typescript
interface TimelineElement {
  id: string;              // UUID
  type: 'ai_transcription' | 'human_memo'; // 要素の種類
  timestamp: number;       // Unixタイムスタンプ（ソート用キー）
  speaker_name: string;    // 「鈴木さん」または「📝人間メモ」
  text: string;            // 発言内容、またはメモ本文
}

```

## 3. バックエンド要件（Tauri / Rust）

### 3.1 メモリキャッシュへの統合（Tauri State）

ユーザーがメモを追記するたびに、TauriのCommand `add_human_memo` を呼び出し、バックエンド側でも時間軸に沿ってメモリ内に保持する。

* **Command名:** `add_human_memo`
* **引数:** `meeting_id: String, text: String, timestamp: u64`
* **処理内容:**
    1. 会議セッション中のメモリバッファ（`Vec<TimelineElement>`）にメモをパースして追加。
    2. 途中でアプリが強制終了した場合の対策として、SQLiteの暫定ログテーブル、または `meetings` テーブルのテンポラリバッファに非同期で即座に保存（永続化）する。



### 3.2 後段タスクへのコンテキスト準備

この手動メモは、単に画面に表示するだけでなく、会議終了後にQwen 35Bが議事録を清書する際の「最も信頼性の高い重要データ」として扱えるよう、データ構造を保ったままFile 5のバッチ処理エンジンへ引き渡せるように設計する。

---

## 4. プログラマーへの実装指示（File 4での実装範囲）

1. **ステップ1 (フロント):** Quasarのタイムライン画面の底部に、固定の `q-input`（送信ボタン付き）を配置する。
2. **ステップ2 (フロント):** メモを送信した際、現在のシステム時刻（タイムスタンプ）を取得し、AIのダミーログが並ぶ配列（`ref<TimelineElement[]>`）の「正しい時間順の位置」にオブジェクトをインサートして画面が再レンダリングされる Vue 3 のロジックを組む。
3. **ステップ3 (バックエンド):** TauriのCommand `add_human_memo` を作成し、フロントから送られてきた手動メモをRust側の変数（セッション状態）にプッシュするロジックを実装する。
4. **ステップ4 (永続化テスト):** メモが送信されるたびに、SQLiteのデータベース内にタイムスタンプとともに正しくレコードが追加されることを確認する。

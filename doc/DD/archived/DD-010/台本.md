# DD-010 評価クリップ 読み上げ台本（2話者）

> 用途: Phase 1 の評価クリップ録音用。**2人で下の台詞を音読**して録音する（DD-003 と同じ音読方式）。
> 正解テキスト（CER用）は話者ラベルを除いた本台本そのもの = [`python/audio/script02.txt`](../../../python/audio/script02.txt)。
> 題材は議事録ツールの進捗会議。**専門用語や略語は使わず、誰でも自然に読める普段の会話**にしてある（音読のしやすさ優先）。

## 配役

- **鈴木**（進行役）
- **佐藤**（報告役）

## 読み方・録音手順

1. `python/` 配下で録音コマンドを実行（無音は自動で除かれるので秒数は長めでよい）:
   ```
   uv run python -m synchroni_note.bench.record_audio --seconds 180 --out audio/sample02.wav
   ```
2. カウントダウン後、**鈴木→佐藤の順で交互に**、普段の会議くらいの自然な速さで音読する。
3. 台詞の順番どおりに読む（飛ばさない・足さない）。少し言い間違えても、そのまま続けて構わない（実際の会議に近い音声になる）。
4. 読み終えたら停止を待つ（180秒以内なら自動停止）。

## 台詞

**鈴木**: お疲れ様です。今日は議事録ツールの進み具合を確認しましょう。佐藤さん、先週からの状況を教えてください。

**佐藤**: はい。録音した音声を自動で文字に起こす部分が、ひとまず動くようになりました。短い音声なら、ほぼ問題なく文字になります。

**鈴木**: それは良いですね。会議の途中でも、話した内容がその場で画面に出る感じですか。

**佐藤**: そうです。まず話した言葉をそのまま表示して、要約は会議が終わってからまとめて作る方針です。途中で重い処理をしないので、表示が遅れにくいです。

**鈴木**: なるほど。誰が話したかの区別はどうしますか。

**佐藤**: 会議中はいったん仮の名前で表示して、後から人が手で直せるようにします。自動の判別は難しいので、無理に会議中にはやりません。

**鈴木**: 分かりました。長い会議だと時間がかかりそうですが、その辺りは大丈夫ですか。

**佐藤**: そこはまだ確認中です。来週までに、実際の会議に近い音声で時間を計って、結果を共有します。

**鈴木**: お願いします。では、今日決まったことを議事録にまとめて、関係者に共有しておきます。

**佐藤**: 承知しました。以上です。

## メモ

- 内容は議事録ツールの進捗会議（決定事項・宿題・期限を含む）。後で要約の評価にも使える素材になっている。
- もし「自分たちの実際の会議でよく出る言葉」を入れたい場合は、台詞を差し替えてOK（その場合は `script02.txt` も同じ内容に合わせて更新する）。

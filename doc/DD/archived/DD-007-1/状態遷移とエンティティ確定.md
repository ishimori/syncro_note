# 状態遷移とエンティティ確定（DD-007-1 Phase 3 成果物）

## 1. 会議ライフサイクル（状態遷移）

```
            S-02 保存して予約
[*] ───────────────────────────▶ scheduled
                                    │ S-04 録音開始（actual_start 記録）
                                    ▼
                                  active ──────┐ クラッシュ → 再起動時 active 残留を検出
                                    │ S-05 会議を終了    │ → timeline_elements の flush 済み行から復旧
                                    ▼            ◀───────┘
                                generating（モデル切替→特大MoE清書ストリーミング）
                                    │ S-07 保存（completed）actual_end 記録
                                    ▼
                                completed ──▶ S-03 で閲覧
```

補助状態:
- **aborted**: 予約のまま開催されない／会議中に破棄。`scheduled`/`active` から手動破棄で遷移（任意）。
- **recovering** は永続 status にせず、`active` 残留をアプリ起動ロジックが扱う**ランタイム概念**とする（enum を増やしすぎない）。

クラッシュ残留の復旧（起動時ロジック・DBスキーマ変更は不要）:
- **active 残留**（録音中クラッシュ）: `timeline_elements` の flush 済み行から復元（確定原文は immutable）。
- **generating 残留**（清書中クラッシュ）: `status='generating'` かつ `final_minutes IS NULL` を検出し、永続済み `timeline_elements` から**清書を再実行**（または手動 `aborted`）。timeline は確定済みなので再生成可能。放置すると S-01 上で「生成中」のまま固着する孤児会議になるため、起動時に必ず処理する。

### 各遷移の read / write
| 遷移 | 主な write | 主な read |
|------|-----------|-----------|
| → scheduled (S-02) | meetings(INSERT,status=scheduled) / participants / vocabularies / attachments（解析後 extracted_text・parse_status 更新） | — |
| → active (S-04) | meetings(status=active, actual_start) | participants/vocabularies/agenda（プロンプト前提） |
| active 中 (S-05) | timeline_elements(逐次flush) / speaker_mappings(確定・推測) | speaker_mappings（表示名導出） |
| → generating (S-05終了) | meetings(status=generating) | meetings/attachments.extracted_text/timeline_elements(human_memo最優先)/speaker_mappings(確定名) |
| → completed (S-07保存) | meetings(final_minutes, status=completed, actual_end, batch_model, generation_seconds) | — |
| 閲覧 (S-03) | — | meetings/participants/attachments/timeline_elements |

> status enum 確定: **`scheduled` / `active` / `generating` / `completed` / `aborted`**（要件の3値に generating・aborted を追加）。

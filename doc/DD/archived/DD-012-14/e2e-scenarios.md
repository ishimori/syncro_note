# DD-012-14 E2Eシナリオ（操作→期待結果）

> 前提→操作手順→期待結果の形式。実マイク依存部は実ウィンドウ(CDP)で確認、見た目は Playwright/CDP のスクショで確認する。

## S1: プリフライトで実マイク一覧が出る
- **前提**: アプリ起動済み。PCに複数の入力デバイスがある。
- **操作**: S-04（プリフライト）を開く → 入力デバイスのプルダウンを開く。
- **期待**: 固定3択ではなく、実機の入力デバイス名が並ぶ。既定デバイスが初期選択（または設定の `mic_device` が選択）。

## S2: デバイスを選ぶとそのマイクで計測される
- **前提**: S-04 表示中。既定とは別のマイクがある。
- **操作**: プルダウンで別マイクを選ぶ → 選んだマイクに向かって話す。
- **期待**: 入力レベルバーがそのマイクの音で動く。ステータスは（DD-012-8 の通り）一度検出したら「入力を検出」で維持。デバイス変更直後は検出状態がリセットされる。

## S3: 選んだマイクが録音にも使われる
- **前提**: S-04 で既定以外のマイクを選択済み。
- **操作**: 「録音開始」→ S-05 で録音 → 選んだマイクに話す。
- **期待**: 文字起こしがそのマイクの音で進む（既定マイクではなく選択マイクから収音）。

## S4: 選択は記憶される
- **前提**: S-04 または S-08 でマイクを選択。
- **操作**: アプリを開き直す／S-08 設定を再表示。
- **期待**: 前回選んだマイクが初期選択になっている（設定 `mic_device` に名前で永続）。

## S5: 抜けたデバイスは既定にフォールバック
- **前提**: 設定に保存したマイク名が、現在の一覧に存在しない（抜いた等）。
- **操作**: S-04 を開く／録音開始。
- **期待**: エラーにならず、OS既定デバイスで計測・録音が続く（フォールバック）。一覧には実在デバイスのみ表示。

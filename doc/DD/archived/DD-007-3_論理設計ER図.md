## 目的

DD-007-1 のエンティティと DD-007-2 の辞書をもとに、**ER図・テーブル責務・関連（FK）・正規化判断**を確定する。物理DDLは書かない（DD-007-4 へ）。

## 背景・課題

[SSOT §8 訂正一覧](../spec/基本設計書.md#L218-L227)を論理モデルに正しく落とす必要がある。特に:
- タイムラインは原文 immutable と LLM整形を分離（[原則1・3](../spec/基本設計書.md#L20-L31)）。
- 話者は confirmed / ai_guess の2層（[SpeakerMap](../spec/基本設計書.md#L96-L102)）。
- `participants.voice_profile` は識別不可 →「名寄せヒント表示用」に役割再定義（[§5](../spec/基本設計書.md#L192)）。

## 検討内容

### 想定エンティティと関連（暫定・DD-007-1の確定で更新）
```
meetings 1 ── N participants
meetings 1 ── N vocabularies
meetings 1 ── N attachments         (extracted_text を保持)
meetings 1 ── N timeline_elements   (ai_transcription | human_memo / 原文 immutable + refined 別カラム)
meetings 1 ── N speaker_mappings     (speaker_id → confirmed / ai_guess)
app_settings        (会議に非従属・単一行)
（スキーマ版管理テーブルは作らない＝マイグレーション不採用 / DD-007-4）
```

### 論点
- speaker_mappings を「行＝(meeting_id, speaker_id)」とし confirmed/ai_guess を列にするか、確定種別を持たせるか。
- human_memo は timeline_elements 内で type 区別か別表か（SSORTは単一timeline・seq順）。SSOT準拠で単一表＋type。
- 正規化レベル（用語/話者の重複許容）と検索要件（カレンダー一覧・議事録詳細の読み取り）。

## 決定事項

成果物 [ER図.md](DD-007-3/ER図.md)（Mermaid＋責務表）＋ [SSOT訂正反映チェック.md](DD-007-3/SSOT訂正反映チェック.md) に集約。要点:
- 7テーブル（meetings 集約ルート＋4子＋timeline_elements＋speaker_mappings、独立 app_settings）。
- **人間メモは別表化せず `timeline_elements.kind` で統合**（seq統合ソートを単純化／SSOT準拠）。
- **speaker_mappings は行=(meeting,speaker_id)・confirmed/ai_guess を列**（一括置換＝1行更新＋UI再導出）。
- 清書バッチ入力は `WHERE meeting_id=?` の1会議スコープで集約可。
- SSOT §8 ＋追加5項目すべて反映先あり（未反映0）。

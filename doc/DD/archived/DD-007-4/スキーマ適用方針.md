# スキーマ適用方針（DD-007-4 成果物）

> 実DDL: [schema.sql](schema.sql)。
> **方針決定（簡素化）**: 過去データの移行は不要（ユーザー確認済み）。よって**マイグレーション機構は作らない**。起動時に `schema.sql` を冪等適用するだけ。版管理（user_version）・差分適用・ロールバック・バックアップは**いずれも不採用**。

## 1. やること（これだけ）

```
アプリ起動
  → DB接続
  → PRAGMA foreign_keys=ON
  → schema.sql を executescript（CREATE TABLE IF NOT EXISTS で冪等）
  → 完了
```

- 初回起動: テーブルが作られる。
- 2回目以降: 既に在るので `IF NOT EXISTS` が素通り（何もしない）。
- これで「初期化」は完結。マイグレーションという段階は存在しない。

## 2. スキーマを変えたくなったら

開発・評価中にスキーマを変更する場合は、**`.sqlite` ファイルを削除して作り直す**（再起動で schema.sql から再生成）。

- 過去データを保持する必要はない、という前提に基づく割り切り。
- そのため `ALTER TABLE` も版番号も不要。schema.sql を編集 → DBファイル削除 → 再起動、で最新定義になる。

> ⚠️ 注意（割り切りの代償）: `CREATE TABLE IF NOT EXISTS` は**既存テーブルがあると変更を反映しない**。schema.sql を編集しても、古い `.sqlite` を消さない限り旧定義のまま。「定義を変えたら DBファイルを消す」をルールとする。

## 3. schema.sql の配置（実装時に移設）

| 期 | ランタイム | 読み込み | 配置 |
|----|-----------|---------|------|
| 評価期 | Python（`sqlite3`） | 起動時 `executescript` | `python/src/synchroni_note/db/schema.sql` |
| 製品期 | Rust（`rusqlite`/`sqlx`） | `include_str!` で埋め込み→適用 | `src-tauri/db/schema.sql` |

- schema.sql を**単一の正**とし両ランタイムで共有。DBファイル実体は `app_settings.db_path`（既定 `app_data_dir`／S-08 表示）。

## 4. 接続時の必須 PRAGMA

| PRAGMA | 値 | 理由 |
|--------|----|------|
| `foreign_keys` | ON | SQLite既定OFF。**接続ごとに必須**（FK CASCADE/SET NULL を効かせる） |
| `journal_mode` | WAL | 書き込みと読みの並行性（SSOT §3.4）。DBに永続するので初回でよい |
| `synchronous` | NORMAL | WAL前提の妥当な耐久性/性能 |
| `busy_timeout` | 5000ms 程度 | 別タスクの flush と競合時の待機（実装時設定） |

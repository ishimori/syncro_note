# データ辞書（DD-007-2 Phase 2 成果物 / 採用値の正）

> [対照表](対照表.md)の食い違いをSSOT優先で収束させた採用値。DD-007-3（論理）/ DD-007-4（物理DDL）はこの辞書に従う。

## 1. 共通規約
| 規約 | 採用値 | 根拠 |
|------|--------|------|
| 文字列ID | `TEXT`（UUID v4） | meetings 等、外部から参照される実体 |
| 子明細ID | `INTEGER PRIMARY KEY AUTOINCREMENT` | vocabularies 等、外部参照されない明細 |
| 真偽値 | `INTEGER`（0/1）。列名は `is_*` / `use_*` / `keep_*` | SQLite に bool 型なし |
| 日時（絶対） | `TEXT`（ISO8601 `YYYY-MM-DDTHH:MM:SS`） | SQLite 慣習・可読・ソート可 |
| 時刻（会議内相対） | `INTEGER`（ミリ秒, 会議 `actual_start` 基点） | SSOT `t_ms` |
| 監査列 | `created_at` / `updated_at`（ISO8601 TEXT） | 全永続テーブルに付与（明細は最小限） |
| 命名 | snake_case・単数形カラム／複数形テーブル | — |
| 列挙 | アプリ層 enum ＋ DB側 `CHECK` 制約で二重防御 | — |

## 2. 列挙値（確定）
| 列挙 | 採用値 |
|------|--------|
| `meetings.status` | `scheduled` / `active` / `generating` / `completed` / `aborted` |
| `timeline_elements.kind` | `ai_transcription` / `human_memo` |
| `attachments.file_type` | `xlsx` / `pdf` |
| `attachments.parse_status` | `pending` / `done` / `error` |
| `app_settings.kv_cache_type` | `f16` / `q8_0` |

## 3. 話者の表現（最重要・SSOT準拠）
| 項目 | 採用値 |
|------|--------|
| `speaker_id` | `INTEGER`（仮ID 0,1,2…）。`"Speaker_0"` は**表示時の整形**であってDB値ではない |
| 表示名の保持 | **DBに焼き込まない**。`speaker_mappings` から都度導出 |
| 導出規則 | `confirmed_name` ?? `ai_guess_name` ?? `"Speaker_" + speaker_id` |
| 確定の優先 | **人間確定（confirmed）＞ AI推測（ai_guess）**（一括置換＝マッピング差し替え。確定原文は不変） |
| 参加者紐付け | `speaker_mappings.confirmed_participant_id` → `participants.id`（ON DELETE SET NULL） |

## 5. スキーマ適用
版管理・マイグレーションは**行わない**（過去データ移行不要）。起動時に schema.sql を冪等適用するのみ。スキーマ変更時は `.sqlite` を削除して再生成（詳細は DD-007-4）。

## 🔬 機械検証
対照表の全行に採用値が1つ確定（未決0）。SQLite 非対応型（bool/真の TIMESTAMP）は INTEGER/TEXT へ写像済み。

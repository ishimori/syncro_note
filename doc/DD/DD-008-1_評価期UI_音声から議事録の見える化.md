# DD-008-1: 評価期UI（ブラウザで音声 → 議事録の見える化）

| 作成日 | 更新日 | ステータス |
|--------|--------|------------|

> アプローチ: 標準（探索的実装）。親 [DD-008](DD-008_バッチMVP_音声から議事録Markdown.md) の動くパイプラインに最小UIを被せる子DD。
> 位置づけ: **評価期の「見える化」**。本番UIは Quasar/Tauri（別途・他DD）。本UIは体験/UX検証用で、製品UIではない。

## 目的

DD-008 で完成した CLI パイプライン（音声→VAD+medium文字起こし→辞書ケバ取り→Ollama要約→議事録Markdown）を、**ブラウザUIから触れる**ようにする。「音声を入れる → ボタン → 議事録が表示される」を最短で実現し、**動く成果を体感**できる状態にする（＋評価期の使い勝手検証）。

## 背景・課題

- ここまで DD-002/003/005/008 と評価・ベンチが続き、**「出来た！」という見える手応えが無い**（ユーザー要望）。最速で見える成果を出す。
- CLI（ターミナル）だと達成感・体験に乏しい。**画面で音声→議事録**が出ると、中核価値が直感的に伝わり、実会議での使い勝手も評価しやすい。
- 完全ローカル要件は維持（UIもローカルサーバ、クラウド送信なし）。

## 検討内容

- **UIライブラリ**: `gradio`（ローカルサーバ・**ブラウザのマイク録音対応**・最小コードで音声入力→Markdown表示）。本番の Quasar/Tauri とは別物（評価期ツール）。
- **再利用**: `pipeline.transcribe / cleaning / summarize` をそのまま呼ぶ（ロジック重複なし）。
- **UX**: 音声（録音 or アップロード）＋会議名/アジェンダ/モデル選択 → 「議事録を作成」→ 議事録Markdown表示＋ケバ取り済み書き起こし（折りたたみ）。`gradio` のジェネレータ出力で進捗（文字起こし中→要約中→完了）を表示。

## 決定事項

- UIを `python/src/synchroni_note/ui/app.py` に gradio Blocks で実装。CLIエントリ `synchroni-note-ui`（pyproject scripts）。
- `127.0.0.1:7860` でローカル起動。要約モデルは UI で選択（既定 gemma4:26b）。
- 本DDは評価期UI。製品UI（Quasar/Tauri）は別DDで、本UIは将来破棄可。

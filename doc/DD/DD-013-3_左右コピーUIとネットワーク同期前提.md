## 目的

左ペイン（AI）→右ペイン（人間メモ）への**セグメント転送 UI**を実装し、将来の複数クライアント対応に向けた**ネットワーク同期モードの前提（Yjs provider 抽象＝差し替え口）**を用意する。

## 背景・課題

- 企画書の「左→右コピー」は協調編集の体験の要。CRDTコア（DD-013-2）の上に、ユーザー操作とそれが整合性を壊さないことの確認を載せる。
- 複数クライアント運用は将来段。本DDは provider を差し替えられる口だけ用意し、単独UIでも回帰しないようにする。

## 検討内容

**スコープイン**
- 「左→右コピー」UI（マウスクリックでセグメントを人間メモへ転送）
- ネットワーク同期モードの前提整備（複数クライアント対応のための provider 抽象）
- コピー操作が CRDT マージ整合性（衝突0）を壊さないことの確認

**スコープアウト（重複回避）**
- 実際の複数クライアント運用・同期サーバ実装（将来段。本DDは前提のみ）
- シードデータの base_date／タイムゾーン分散 → DD-012-3-1 が持つ（複数ユーザー時の論点は未決事項）
- 清書・保存 → DD-012-2

## 決定事項

- provider は抽象化して差し替え可能にする（ローカル単独でも動く）。

> 位置づけ: DD-011 Phase 3-C（サンプル音声ファイルでの配線）の「マイク差分」。土台は 3-C を再利用し、入力源と連続収音の操作だけを足す。

## 目的

DD-011 Phase 3-C で「サンプル音声ファイル→S-05 に逐次表示」の配線（Rust中継／画面 listen／ダミー撤去／子プロセス kill）が通る。本DDはその入力源をサンプルファイルから**実マイク**へ差し替える差分。マイクで話した内容が S-05 に1行ずつ流れる状態にする。

## 背景・課題

- DD-011 Phase 3 は 3-A（画面シェル・完了）/3-B（Python実行口 sidecar.py・ファイル入力）/3-C（Rust中継＋S-05ライブ配線・ファイル入力）に分割済み（別セッション管轄）。配線の土台（Rust spawn→stdout中継→emit→kill、画面の listen→timeline.push、ダミー撤去）は **3-C が作る**。
- Python 側はマイク収音＋8〜12秒VADチャンク＋逐次STTが DD-010 で実装済み（`python/src/synchroni_note/realtime/`）。
- よって本DDは新規発明ではなく、**3-C の土台に「マイク入力」と「連続収音の操作（開始/一時停止/停止）」を足すだけ**。サイドカー契約（JSON Lines: `meta`/`segment`/`done`/`error`）は不変なので、画面・Rust中継はほぼそのまま再利用できる。

## 検討内容

- **3-C を再利用**: Rust の中継・kill、画面の listen・timeline.push・ダミー撤去は 3-C 成果をそのまま使う（本DDで作り直さない）。
- **差分は3点**: ① sidecar.py に mic モード（DD-010 の realtime 経路を JSON Lines で流す。3-B のファイル版に追記）② Rust の起動をファイル／mic で切替（spawn 引数の差）③ 画面に連続収音の操作（録音 開始/一時停止/停止）。ファイルは1回読み切り、マイクは止めるまで流れ続ける点が主な違い。
- マイクデバイスは S-04/S-08 の選択値を使用（無ければ既定デバイス）。

## スコープ

- **やる**: sidecar.py の mic モード追加、Rust の mic 起動対応、S-05 の録音 開始/一時停止/停止、実マイクでの逐次表示。
- **やらない（＝ DD-011 3-C 担当・再利用）**: Rust中継の新規実装・画面 listen 基盤・ダミー4件の撤去・子プロセス kill の新規実装。
- **やらない（別の子DD）**: 終了後の要約清書（012-2）／保存（012-3）／LLM追い上げ整形（012-4）／話者ラベル（012-5）。

- 流用: **DD-010**（realtime 収音＋逐次STT）／**DD-011 3-B**（sidecar.py）。

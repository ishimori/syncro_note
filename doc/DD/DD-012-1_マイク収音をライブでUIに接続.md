## 目的

DD-011 Phase3（サンプル音声→S-05へ逐次表示）の入力を、**サンプルファイルから実マイクへ**差し替える。マイクで話した内容が S-05 タイムラインに1行ずつ流れる状態にする。

## 背景・課題

- Python 側は DD-010 で「マイク収音＋8〜12秒VADチャンク＋逐次STT（RTF 0.527・体感約4秒）」が実装済み（`python/src/synchroni_note/realtime/`）。
- DD-011 Phase3 で「逐次セグメント→JSON Lines→Rust中継→S-05 push」の経路が確立（予定）。本DDは**その入口を realtime 経路に差し替えるだけ**で、新規発明は少ない。

## 検討内容

- サイドカー契約（[DD-011 Phase3 §3](DD-011/Phase3_実装前詳細化.md) の JSON Lines）はそのまま流用。`meta`/`segment`/`done`/`error` の型は不変。
- 開始トリガー: マイクデバイスは S-04/S-08 の選択値を使用（無ければ既定デバイス）。
- 終了/一時停止での子プロセス kill（DA#4）は Phase3 の方針に従う。

## スコープ

- **やる**: マイク収音→VADチャンク→逐次STT→S-05へ逐次表示。録音開始（S-04「録音開始」/S-05）・一時停止・終了の最小制御。
- **やらない**: 終了後の要約清書（012-2）／保存（012-3）／LLM追い上げ整形（012-4）／話者ラベル（012-5）。

## 依存

- 前提: **DD-011 Phase3 完了**（サイドカー経路）。
- 流用: **DD-010**（realtime 収音＋逐次STT）。

## 目的

会議を保存した後の **S-03「元タイムライン（証跡）」に話者を表示し、人ごとに色分け**する。現状は保存時に話者情報を捨てている（`speakerId: null` 固定）ため、S-03 は古い会議も新しい会議も全行「話者？」のグレー1色になっている。

## 背景・課題

- DD-012-5 で**ライブ（S-05）画面**は話者表示・クリック確定・人ごと自動配色まで実装済み。だが**保存→S-03 への配線が無い**。
- 保存経路 [S05Realtime.vue `endMeeting`](../../app/src/pages/S05Realtime.vue) は `speakerId: null` を書き込み、[session.ts `TimelineRow`](../../app/src/session.ts) も「当面 null」。
- [S03MinutesDetail.vue](../../app/src/pages/S03MinutesDetail.vue) の `speakerName` は `話者 ${speaker_id ?? "?"}` で、色は固定 `grey-2`。
- **DBスキーマは既にこの機能向けに設計済み**: `timeline_elements.speaker_id`(INTEGER) ＋ `speaker_mappings`(meeting_id, speaker_id → confirmed_name/ai_guess_name。表示名導出 `confirmed_name ?? ai_guess_name ?? 'Speaker_'||id`)。**ただし `speaker_mappings` はアプリ側のコードが1行も無い**（CRUD未実装）。

## 検討内容

- 話者ラベル変換: ライブの `speaker`="spk{n}" → `timeline_elements.speaker_id` = 整数 n。末尾数字を抽出（"Speaker_0"/分離なしは 0 or null）。
- 人間確定の保存: S-05 の `mapping[spk]`（確定名）を `speaker_mappings.confirmed_name` に保存。`ai_guess_name` は当面未使用（必要なら後続）。
- S-03 表示名: `confirmed_name ?? ai_guess_name ?? 話者{id}`（スキーマ例の `'Speaker_'||id` は日本語UI向けに `話者{id}` とした。決定事項参照）。
- 色分け: `speaker_id` でパレット選択（S-05 の `speakerColor` と**同じ番号→色**規則。S-05=濃色/S-03=淡色で同系色）。

## 決定事項

- **保存コマンドは既存を拡張**（別コマンドを作らない）: `create_meeting`/`complete_meeting` に `speaker_mappings: Vec<SpeakerMapping>` 引数を追加し、timeline と同じ1トランザクションで書込み。`complete_meeting` は `delete_speaker_mappings → insert` で全入替（timeline と同じ方針）。
- **表示名解決は FE（S-03）で実施**: `get_meeting_detail` は `speaker_mappings` を生のまま返し、`S03MinutesDetail.vue` の computed `speakerNames` で `confirmed_name ?? ai_guess_name ?? 話者{id}` を導出。BE は解決済み文字列を持たない（表示ロジックを FE に一元化）。
- **表示名フォールバックは `話者 {id}`**（schema 例の `'Speaker_'||id` ではなく日本語UIに合わせる）。話者なし（speaker_id=null＝旧データ/人間メモ）は「話者 ?」のまま非破壊。
- **speaker_mappings は確定名があるものだけ保存**（未確定話者は行を作らず S-03 で `話者 {id}` にフォールバック）。`ai_guess_name`・`confirmed_participant_id` は当面 null（参加者エンティティ連携は後続）。
- **配色**: `speaker_id` の番号→パレット。S-05=濃色バッジ（`-6`系）／S-03=淡色気泡（`-2`系）で**同じ番号は同系色**にして画面間で一貫。

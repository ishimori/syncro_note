# DD-015 E2Eシナリオ（「正解」の定義・ユーザーレビュー対象）

> 形式: 前提条件 → 操作手順 → 期待結果。合意後にテスト化／実装着手。

## シナリオA: 今すぐ録音（予定なし・新規）★今回の主目的

- **前提**: アプリ起動済み。予定は作っていない。
- **操作**:
  1. 左メニューの「今すぐ録音」をクリック
  2. プリフライト画面が開く（マイク確認）
  3. 「録音開始」を押す
  4. しゃべる → 文字起こしが出る
  5. 「会議を終了」を押す → 生成 → 保存
- **期待結果**:
  - 手順2でプリフライトのタイトルが「その場で録音」（特定の予定に紐づかない／URLに id が付かない）
  - 手順5の保存後、議事録が **「録音メモ MM-DD HH:MM」** のタイトルで新規作成される
  - カレンダー／履歴一覧にその録音メモが現れる

## シナリオB: 製品ビルドのメニュー（最終形の見た目）

- **前提**: 製品ビルド（`import.meta.env.DEV = false`）。
- **操作**: 左メニューを開く。
- **期待結果**:
  - 表示は **「カレンダー」「今すぐ録音」「設定」の3つだけ**
  - DEV バッジも WIP バッジも**一切出ない**（クリーンな見た目）
  - S-02／S-03／S-04／S-05／S-06／S-07 の生リンクは**出ない**

## シナリオC: 開発ビルドのメニュー（デバッグ導線）

- **前提**: 開発ビルド（`tauri dev`／`import.meta.env.DEV = true`）。
- **操作**: 左メニューを開く。
- **期待結果**:
  - 常設3項目（カレンダー／今すぐ録音／設定）に加え、
  - **S-02〜S-07 の生リンク**が DEV バッジ付きで並ぶ（各画面へ直接ジャンプできる）
  - **WIP バッジはどこにも出ない**（全廃）

## シナリオD: 既存フロー回帰（繋ぎが壊れていないこと）★ミス防止の要

- **前提**: カレンダーに予定（completed／進行中／未開始）がある。
- **操作と期待結果**（メニュー改修後も従来どおり動くこと）:
  | 操作 | 期待遷移 |
  |------|---------|
  | カレンダーで completed の予定をクリック | 議事録詳細（S-03）が開く |
  | カレンダーで 進行中・生成中 の予定をクリック | リアルタイム（S-05）がその予定id付きで開く |
  | カレンダーで 未開始/その他 の予定をクリック | 会議作成（S-02）が開く |
  | 予定から S-04「録音開始」 | S-05 がその予定id付きで開く（録音→保存でその予定が「完了」に書き戻る） |
  | S-05「会議を終了」 | S-06 生成 → 保存で S-07 プレビュー |
  | 録音中に別画面へ移動 | マイクが停止する（録音が宙ぶらりんにならない） |
  - **要点**: 予定に紐づく録音は従来どおり「その予定」に保存され、今日の日付にズレない（DD-012-12 の挙動を維持）。

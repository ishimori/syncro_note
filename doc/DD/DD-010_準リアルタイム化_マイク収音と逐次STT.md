## 目的

マイク入力から **8〜12秒VAD区切り**でチャンクを切り出し、逐次 faster-whisper にかけて**確定テキストを時系列表示**する経路を評価期Pythonで初めて通す。中核設計（[基本設計書 原則1: 確定テキスト即表示が主役](../spec/基本設計書.md)）のリアルタイム経路を実装し、**体感遅延・取りこぼし率・RTF を実測**して Phase 2 の合否を判断する。

## 背景・課題

- Phase 1（バッチMVP, [DD-008](DD-008_バッチMVP_音声から議事録Markdown.md)）が一次達成。次は収音→逐次STT（準リアルタイム）。
- **リアルタイム可否ゲートは実測済み**: faster-whisper medium の 8〜12秒VAD streaming で **RTF 0.43〜0.61（<1）**（[DD-003結果](DD-003/結果.md)）。本DDは「速度が出るか」ではなく**「逐次経路を通して体感遅延・取りこぼしが実用域か」**を確かめる段。
- **評価音声が不足**: 現状 sample01（70秒・単一話者）のみ。**複数話者・専門用語を含む代表クリップ**が必要（取りこぼし率/CER測定の前提）。これは [DD-008結果](DD-008/結果.md)「実会議・複数話者は未評価」という Phase 1 の穴も同時に埋める。

## 検討内容（設計の要点・証拠付き）

- **チャンク化方式**: 固定3秒チャンクは medium で RTF 1.39〜1.78 と破綻（30秒窓パディング）。→ **無音主体の 8〜12秒VAD区切り**を採用し RTF 0.43〜0.61 に回復（[DD-003結果](DD-003/結果.md)・[基本設計書 §3.4/§7](../spec/基本設計書.md)）。
- **STTエンジン**: 評価期 faster-whisper medium（CER 0.26・RTF~0.5）を基準。即時性を上げる **二段STT（base即時＋medium大窓）** は base streaming RTF 0.098 と余裕があり有力だが、本DDは **medium単段を基準・二段は stretch**（[基本設計書 §7](../spec/基本設計書.md)）。
- **同時実行**: 本DDは **STT単体**（LLM要約は P2-3 別DD）。時間分離問題（whisper↔Ollama競合, [DD-003](DD-003/結果.md)）は本DDでは顕在化しないため経路がシンプル。
- **再利用（Phase 0 で精査済み）**:
  - チャンク化 = `bench/vad_segment.py`（`detect_voiced_spans`／`split_long(max_seg_s≈10)`・純関数pytest済）。**アルゴリズムは再利用、ただし全体配列前提**なのでライブ用に**ローリングバッファへの逐次適用ラッパが新規**。
  - STT・指標 = `bench/stt_bench.py`（`_load_model`／`_transcribe`／`real_time_factor`／`character_error_rate`／`_load_audio`）。
  - ⚠️ `bench/record_audio.py` の `record()` は `sd.rec`＝**固定長バッチ録音でストリーミング不可**。ライブ収音は **`sd.InputStream` コールバックを新規実装**（再利用は `to_int16_pcm`／`write_wav`／デバイス列挙のみ）。Phase 1 のクリップ録音には `record_audio.py` をそのまま使用。
- **出力**: ターミナル簡易表示（確定セグメントを `seq` 順・遅延/ドロップを併記）。UIは持たない。
- **取りこぼし率の測り方（精査で確定）**: streaming（VADチャンク逐次）の連結出力 vs batch（全体一括）出力の **CER差分**で境界（seam）コストを定量化（`stt_bench` が両モード対応済みで比較可能）。加えてキュー満杯時の **drop計数**。
- **揮発メトリクス**: 体感遅延（チャンク確定時刻 − 発話終了時刻の近似）・取りこぼし率（上記CER差分＋drop数）・RTF。[基本設計書 §3.4](../spec/基本設計書.md) の遅延ゲージ/ドロップ計数のPython簡易版。

## 決定事項

- 新規 `python/src/synchroni_note/realtime/` パッケージ（収音ループ＝`sd.InputStream`＋チャンクキュー＋逐次STT）。CLIエントリ `synchroni-note-stream`（pyproject）。
- チャンク = **8〜12秒VAD区切り**（`vad_segment` アルゴリズム＋ローリング適用）。エンジン = **faster-whisper medium（既定）**。二段STTは stretch。
- 指標は `stt_bench` の RTF/CER 関数を再利用。取りこぼし率＝**streaming連結 vs batch の CER差分**＋drop数。
- 評価 = **新規録音の代表クリップ（複数話者・専門用語）**で 体感遅延 / 取りこぼし率 / RTF を実測（[ロードマップ §5 指標](../plan/開発ロードマップ.md)）。
- **P2-3 ローリング要約は別DD**。

## 目的

このPC（Ryzen 7 PRO 8840HS / CPU駆動 / RAM制約あり）で、ローカルLLMが**議事録要約タスク相当**をどれくらいの速度（tok/s）・レイテンシで処理できるかをモデルサイズ別に実測し、**要約モデルを決定する材料**を得る。

## 背景・課題

- ロードマップ最大リスク R1「CPUで32Bが要約に追いつかない」/ R2「利用可能RAMが現状9.6GBで32Bが載らない」が未検証。
- 企画書は `qwen3:32b` を本命に置くが、CPUでの実用速度・メモリ収まりは実測しないと判断できない。
- 要約が「リアルタイム追従」か「トピック区切りバッチ」かは、ここで得る tok/s 次第。

## 検討内容

**対象モデル**
- ベースライン: `qwen3:8b` / `qwen3:14b`、メモリ確保後に `qwen3:32b`。量子化はOllama既定（概ね Q4_K_M）。
- 着手時に `ollama list` と公式レジストリで、より新しい系列（例: qwen3.5系が利用可能なら）も候補確認。

**計測上の注意（重要）**
- qwen3 はハイブリッド思考モデル。要約用途では**思考オフ（`/no_think` 相当 / `think=false`）で計測**する（思考トークンが速度・レイテンシ・出力を歪めるため）。比較目的で思考オンも任意で取得。

**計測指標**（Ollama `/api/generate` のメトリクスを利用）
- 生成速度 tok/s = `eval_count / (eval_duration[ns] / 1e9)`
- プロンプト処理速度 tok/s = `prompt_eval_count / (prompt_eval_duration / 1e9)`
- TTFT（最初のトークンまでの体感遅延、ストリーミングで測定）
- 総処理時間、ピークRAM使用（任意・OS側で観測）

**プロンプト（固定サンプル）**
- 数百〜千トークン規模の日本語「書き起こし風テキスト」を固定で用意し、議事録Markdown（要約/決定事項/TODO）へ要約させる。

**試行**: ウォームアップ1回 + 計測3回の平均。

**実装方針**
- Ollama REST API を Python から叩く。依存は公式 `ollama` パッケージをランタイム依存に追加（薄く扱いやすい）。
- tok/s 算出は純関数に切り出し pytest で検証。API呼び出し部はベンチスクリプト本体。
- 結果は `doc/DD/DD-002/結果.md`（添付）に表で記録。

## 決定事項

- ベンチコードを `python/src/synchroni_note/bench/llm_bench.py` に実装、CLIエントリで実行。
- ランタイム依存に `ollama`（公式client）を追加。
- 計測は思考オフを基本とする。
- 結果と考察（要約モデル候補の暫定判断）を添付に残し、ロードマップ §8 のオープン論点「要約モデル確定」へ反映。

### 2026-06-07
- DD作成
- 要約モデル候補メモ（別セッションのClaude調査より追記。PH3計測中に判明）:
  **Gemma 4 QAT / MoE（2026-06リリース）を Qwen3 の比較候補に追加検討**
  - QAT = 量子化を学習時にシミュレートし、4bitでも品質劣化が小さい手法（Gemma 3 QATで27BがVRAM 54→14.1GBの実績。Gemma 4もQATチェックポイント提供）。
  - **26B A4B MoE** = 総26B/活性3.8B。CPU推論の律速はメモリ帯域で、毎トークン読むのは活性分のみ→「26B級の品質を4B級の速度で」。GPUなし・AVX-512の本機と好相性。
  - メモリ目安(Q4): `gemma4:26b` ~14–18GB / `gemma4:e4b` ~5GB。RAM 59.7GBなら26B MoEはSTT同居でも収まる見込み。
  - 追加候補: `ollama pull gemma4:26b` / `ollama pull gemma4:e4b`（思考オフ等の計測条件はQwenと揃える）。検討内容の「新系列の候補確認」に該当。
  - Ollama側の節約: 長コンテキスト向けKVキャッシュ量子化 `OLLAMA_KV_CACHE_TYPE=q8_0`（メモリ半減・劣化ほぼなし）。
  - 注意: サイズ/メモリは二次情報を含む暫定値。**tok/sと日本語要約品質は本DDのベンチで実測して確定**すること。
  - 出典: Google Developers Blog(Gemma3 QAT) / Google Blog(Gemma4 QAT) / Aurigaiガイド / ModelPiper(Ollama KVキャッシュ)。
- Phase 3 計測実行・完了:

  - **gemma4:26b が e4b 並みに速い**ことを実証（「26B級品質を4B級速度で」）。終了後バッチ清書の本命に。

## 目的

AI 側の自動追記と人間のキーボード入力が並行してもテキストが壊れず**順序が保証される** CRDT マージを実装し、**500字以上の連続編集で衝突0・CRC一致**を検証する。DD-013 の中核。

## 背景・課題

- DD-013-1 の土台（Yjs＋mock AI）の上に、実際のマージ整合性とその検証を載せる。
- 基本設計書: マージ遅延は**画面再描画込みで 200ms 以下**（ちらつき・巻き戻り回避）。テストは素テキスト（半角・記号なし）で original vs merged の CRC 一致を確認。

## 検討内容

**スコープイン**
- TimelineElement の Yjs 統合版データ構造定義（AiTranscription / HumanMemo を CRDT 上で扱う）
- AI追記フロー（emit→Yjs doc append）と人間入力フロー（input→local Yjs update）のマージ
- マージ競合の検出・ロギング
- 素テキスト 500字以上（連続追記 vs 分散キータイプ）での衝突0・CRC一致テスト（unit＋integration）
- マージ遅延を画面再描画込み 200ms 以下に保つ

**スコープアウト（重複回避）**
- broadcast channel 方式（基本設計書 §3.2 で採用禁止）
- 実 Rust whisper からの token 流入経路 → DD-014-4
- ネットワーク同期／複数クライアント → DD-013-3
- 確定テキストの可変化（確定は immutable、整形は別レイヤで隔離）

## 決定事項

- マージ正当性はテスト（CRC一致・衝突0）で機械検証する。
- 遅延予算 200ms（再描画込み）を SLA とする。

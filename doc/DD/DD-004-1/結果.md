# DD-004-1 結果

## Phase 0–1: 本命手法（ONNX話者埋め込み）の実装と疎通（2026-06-07）

### オフライン制約の実地確認（DA#1 = 最重要）→ PASS

- 採用モデル **3D-Speaker CAM++**（`campplus_sv_zh-cn_16k-common.onnx`, 約27MB, 192次元埋め込み）は **sherpa-onnx リリース配布・Apache-2.0・HuggingFace gate なし**。
- ネット到達を実測（GitHub/HF とも HTTP 200）→ **初回DLのみネット要、DL後は完全オフライン**で推論可能。`curl` で直接取得でき、トークン・規約承認は不要。
- → **完全オフライン要件をクリア**。pyannote（HF gated・初回トークン要）を採らずに本命が成立。

### 依存統合で判明した障害（実地の重要知見）

- `sherpa-onnx`(python) 1.13.2 の wrapper は **自前同梱の ONNX Runtime(1.17.1) と API バージョン不整合**（API24要求 vs 17まで）で `SpeakerEmbeddingExtractor` が起動不可。共有venvの `onnxruntime 1.26.0` 単体は正常（faster-whisper も無影響）。
- → **sherpa-onnx wrapper を使わず**、`onnxruntime 1.26.0` で CAM++ ONNX を直接実行し、特徴抽出のみ軽量な **`kaldi-native-fbank`**（CAM++学習時と同じKaldi 80次元fbank, torch不要）に置換。共有venvを壊さず成立。
- 構成: VAD区間 → knf fbank(80) → 発話内CMN → CAM++(onnxruntime) → 192次元L2正規化埋め込み → numpy KMeans(k=2)。

### ベンチ結果（sample02.wav, 120s, 参照=DD-004のsample02.rttm, collar=0.25s）

| method | DER | conf | spk_hyp | RTF | 備考 |
|--------|-----|------|---------|-----|------|
| dummy | 0.380 | 0.380 | 1 | ~0 | 最弱基準 |
| dummy-alt | 0.300 | 0.300 | 2 | ~0 | 交互決め打ち |
| simple-cluster（MFCC, DD-004） | **0.036** | 0.036 | 2 | 0.001 | 手作り特徴baseline |
| **onnx-embed（CAM++埋め込み, 本命）** | **0.047** | 0.047 | 2 | 0.008 | 訓練済み埋め込み |

### 解釈（誇張回避・重要）

- **本命 onnx-embed(0.047) は baseline simple-cluster(0.036) を上回らなかった**。ただし両者とも ~95% 精度で、差 ~1% は35区間中1〜2区間＝**ノイズレベルで実質互角**。
- 原因は素材が**易しすぎる**こと（無オーバーラップ・2話者固定・別人・クリーン）。この条件では手作りMFCCでも飽和し、埋め込みの優位が出ない。**この素材では優劣を判定できない**。
- 埋め込み手法の本領（雑音・声の被り・話者数増・似た声でのロバスト性）は **被り有り/3〜4話者の素材**でしか測れない → ⏸️後回し中の追加素材が**採用確定の前提**。
- 補足: CAM++ は中国語学習モデルだが日本語2話者を分離できた（話者埋め込みは言語非依存性が高い）。多言語/VoxCeleb系モデルの比較は追加候補。

# DD-004-1 評価クリップ 読み上げ台本（4話者）

> 用途: DD-004-1 の**話者分離・人数増テスト**用クリップ（sample03）の録音台本。
> 話者分離が **3〜4人**でどれだけ精度を保てるかを測るため、**4人で下の台詞を音読**して録音する（sample02 と同じ音読方式）。
> 正解データ（誰がいつ話したか）は本台本と1字一句一致する [`python/audio/script03_turns.json`](../../../python/audio/script03_turns.json)。録音した `audio/sample03.wav` を置けば、それを使って自動採点できる。
> 題材は議事録ツールの進捗会議。**専門用語や略語は避け、誰でも自然に読める普段の会話**にしてある（音読のしやすさ優先）。

## 配役（4人・できれば声の違う人で）

- **鈴木**（進行役）
- **佐藤**（文字起こし担当）
- **田中**（要約担当）
- **山本**（話者識別担当）

> ⚠️ 評価の前提として、**4人はそれぞれ別の人**が読むこと（声が別人であることが話者分離の評価の前提）。
> 4人そろわない場合は3人でもよいが、その場合は台詞の担当を3人に割り直し、`script03_turns.json` も同じ割り当てに直す（私に言ってくれれば直します）。

## 読み方・録音手順

1. `python/` 配下で録音コマンドを実行（無音は自動で除かれるので秒数は長めでよい）:
   ```
   uv run python -m synchroni_note.bench.record_audio --seconds 240 --out audio/sample03.wav
   ```
2. カウントダウン後、**台詞の番号順に、担当者が交代しながら**、普段の会議くらいの自然な速さで音読する。
3. **一度に話すのは1人だけ**（声を重ねない）。次の人は前の人が言い終わってから読む。話者の切れ目に**軽く一拍おく**と、後の処理がきれいになる。
4. 順番どおりに読む（飛ばさない・足さない）。少し言い間違えても、そのまま続けて構わない（実際の会議に近い音声になる）。
5. 全部読み終えたら、240秒の自動停止を待つ（早く終わってもそのままでOK／余った無音は自動で除かれる）。

> マイク権限が必要。入力デバイスを選びたいときは `--list-devices` で番号を確認し `--device <番号>` を付ける。

## 台詞

**鈴木**: みなさん、お疲れさまです。今日は議事録ツールの進み具合を、担当ごとに順番に確認していきます。まず佐藤さんから、文字起こしの状況をお願いします。

**佐藤**: はい。録音した音声を文字にする部分は、短い音声ならほぼ問題なく動いています。長い会議でも止まらないように、少しずつ区切って処理するようにしました。

**鈴木**: ありがとうございます。精度の方はどうですか。

**佐藤**: 静かな環境ならかなり正確です。ただ、雑音が多い場所だと取りこぼしが増えるので、そこは来週もう少し試します。

**鈴木**: わかりました。では次に、要約の担当の田中さん、お願いします。

**田中**: はい。会議が終わったあとに、全体を読みやすい議事録にまとめる部分を作っています。決まったことと、宿題と、誰がいつまでにやるかを、自動で箇条書きにします。

**鈴木**: それは助かりますね。今のところ、まとめの質はどうですか。

**田中**: 短い会議なら十分使えます。長くて話が行ったり来たりすると、たまに要点が抜けるので、そこを改善中です。来週、実際の会議に近い長さで試してみます。

**鈴木**: お願いします。続いて、話者の聞き分けを担当している山本さん、どうですか。

**山本**: はい。今は、声の特徴から誰が話したかを後でまとめて振り分ける仕組みを試しています。二人の会話なら、かなり正確に分けられました。

**鈴木**: 三人以上だとどうなりそうですか。

**山本**: そこがこれからの確認点です。人数が増えたり、声が似ていたりすると難しくなるので、今日のこの録音を使って試すつもりです。

**田中**: 山本さん、話す人が途中で重なったときはどうなりますか。

**山本**: 重なると今の仕組みでは弱いです。まずは順番に話す前提で精度を見て、そのあと重なりに強い方法を考えます。

**佐藤**: 文字起こしの側でも、誰が話したかが分かると、議事録がぐっと読みやすくなりますね。

**田中**: そうですね。担当付きの宿題リストも自動で作りやすくなります。

**鈴木**: では今日決めたことを確認します。佐藤さんは雑音の多い音声を来週試す。田中さんは長い会議でのまとめを来週試す。山本さんはこの録音で三人以上の聞き分けを試す。これで進めましょう。

**山本**: 承知しました。

**佐藤**: 了解です。

**田中**: わかりました。以上ですね。

**鈴木**: はい、ありがとうございました。今日の内容は議事録にまとめて、あとで共有します。

## メモ

- 全21ターン・4話者。各担当が複数回話すので、話者ごとの声サンプルが十分に取れる。
- 内容は進捗会議（決定事項・宿題・期限つき）。後で**要約の評価**にも使える素材になっている。
- 録音できたら `python/audio/sample03.wav` に置く → 私の方で正解(RTTM)を自動生成し、`onnx-embed` と `simple-cluster` を4話者で比較する（人数指定 k=4／話者数の自動推定もこの素材で評価）。
- 内容を自分たちの実際の言い回しに差し替えてもOK。その場合は `script03_turns.json` も同じ内容に合わせて更新する（私に言ってくれれば直します）。

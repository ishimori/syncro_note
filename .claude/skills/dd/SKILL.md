# DD設計書操作

DD（Design Document）設計書を操作するスキルです。

---

## ⚠️ 重要ルール（CRITICAL）

### アーカイブ時の必須確認

**移動前に必ず実行:**
```bash
ls doc/DD/ | grep DD-{番号}
```

**フォルダが存在すれば、本体（.md）と一緒に移動すること。**

| 移動元 | 移動先 |
|--------|--------|
| `doc/DD/DD-{番号}_*.md` | `doc/DD/archived/` |
| `doc/DD/DD-{番号}/` | `doc/DD/archived/DD-{番号}/` |

**チェックリスト（必須）:**
- [ ] DD本体（.md）を移動した
- [ ] DD番号フォルダが存在する場合、移動した
- [ ] 元の場所にDD関連ファイルが残っていない
- [ ] `bash scripts/dd-index-gen.sh` を実行してDD-INDEX.mdを再生成

**チェックリスト未確認でのアーカイブ完了報告は禁止。**

- 「DD-027をアーカイブ」 → アーカイブ
- 「DD-080の添付ファイルを作って」「関連資料を追加」 → 添付ファイル作成

## DD番号の解釈（柔軟に）

以下はすべて同じDD-001を指す：
- `DD-001` / `DD 001` / `DD001` / `001` / `1`

識別子付き（チーム開発時）：
- `DDI-001`（石森）、`DDSA-001`（斉藤）など

## 操作手順

### 参照

1. `doc/DD/DD-INDEX.md` が存在すればINDEXからDD番号→ファイル名を解決。存在しなければ `doc/DD/DD-{番号}_*.md` をファイルスキャンで検索（なければアーカイブも検索）
2. **DD本体のみ読み込む**（リンク先は自動で読み込まない）
3. DD番号フォルダ（`DD-{番号}/`）があれば通知
4. 「何をしますか？」と確認

### 新規作成

1. **`doc/templates/guides.md` を Read** → アプローチと使用するテンプレートを決定
2. **`doc/templates/dd_template.md`（+ 差分テンプレート）を Read** → DD骨格を取得
3. `doc/DD/DD-INDEX.md` が存在すればINDEXから最大番号を取得。存在しなければファイルスキャン（`ls`）で取得
4. 次の連番でファイル作成: `DD-{連番}_{タイトル}.md`
5. テンプレートの `{番号}` `{タイトル}` `{日付}` を置換して作成
6. **出力するDDにテンプレートのHTMLコメント（`<!-- ... -->`）を残さないこと**
7. **Playwright MCP 確認**（画面を伴う実装Phaseがある場合のみ）: Playwright MCP が利用可能か確認する。利用不可の場合、DDログに「Playwright MCP 未導入: エビデンスは手動キャプチャで代替」と記載する
8. **DD-INDEX.md 更新**: `bash scripts/dd-index-gen.sh` を実行してインデックスを再生成する。スクリプトが存在しない場合はユーザーに通知し、手動でのインデックス更新を提案する

> **⛔ テンプレート必須ルール**: 上記ステップ1・2 の Read に失敗した場合は**エラーとして停止**し、ユーザーにパスを確認すること。自力でDD構造を生成してはならない。

### 一覧

1. `doc/DD/DD-INDEX.md` を Read して内容を表示
2. DD-INDEX.md が存在しない場合は `bash scripts/dd-index-gen.sh` を実行して生成。スクリプトも存在しない場合はファイルスキャンで一覧を表示

### 検索

キーワードでDDを横断検索する。

1. **Phase 1: インデックス検索（高速）**
   - `doc/DD/DD-INDEX.md` を Read し、テーブル行からキーワードにマッチするDDを抽出
   - Phase 1 で見つかれば結果表示 →「全文検索もしますか？」と確認
   - Phase 1 で 0件なら自動的に Phase 2 へ

2. **Phase 2: 全文検索（必要に応じて）**
   - Grep で `doc/DD/`（アーカイブ `doc/DD/archived/` 含む）の `.md` ファイルを検索
   - 10件超はタイトルマッチを優先表示

### インデックス再構築

`bash scripts/dd-index-gen.sh` を実行して DD-INDEX.md を全量再生成する。スクリプトが存在しない場合はユーザーに `templates/scripts/dd-index-gen.sh` のコピーを提案する。

### ログ追記

1. 対象DDの「ログ」セクションに日時とメモを追記
2. フォーマット: `### {日付}` の下に `- {メモ内容}`

### アーカイブ

→ **「⚠️ 重要ルール」セクションの手順に従うこと**
